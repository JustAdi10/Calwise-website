Privacy Policy

This Privacy Policy describes how Calwise handles your information based on the current iOS app implementation.

1. Information We Collect

Calwise may process the following categories of data when you use the app:

  • Account and profile data: name, email, optional phone, age, gender, height, current weight, target weight, goals, and onboarding preferences.
  • Nutrition and progress data: diary entries, saved meals, macro totals, weight history, streak/progress metrics, and app settings.
  • Feature input data: food text prompts, voice-transcribed text, food images, and barcode values when you use those features.
  • Operational data: API usage metadata such as endpoint, success/failure, request id, and usage cost fields for nutrition-related requests.

2. Where Data Is Stored

Calwise uses both on-device storage and cloud storage.

  • On-device: app state and preferences (UserDefaults), plus local cache storage for foods and diary entries (Core Data/SQLite).
  • Cloud: Supabase authentication and database tables for account, profile, settings, food logs, saved meals, progress, and related records.

3. Third-Party Services and APIs

Calwise integrates with third-party providers for specific features:

  • Supabase: authentication, session handling, and app data sync.
  • USDA FoodData Central: food search and nutrition lookup.
  • Open Food Facts: barcode-based product lookup.
  • Google Gemini API and/or OpenAI API: food analysis from text or images, and serving/nutrition estimation features.
  • Apple services: Sign in with Apple and iOS Speech Recognition when voice logging is used.

The iOS project currently includes Firebase packages as dependencies, but current runtime app code does not initialize Firebase Analytics/Crashlytics flows.

4. Health Data (HealthKit)

The app currently has an Apple Health toggle in settings, but active HealthKit read/write integration is not implemented in the current code path, and no HealthKit entitlement is configured in the app entitlement file.

5. Data Retention

Retention is currently driven by app behavior and backend persistence:

  • Food cache entries on device are evicted after about 30 days.
  • Legacy cached search/offline operation data uses a 7-day expiry window.
  • Unsynced offline entries may remain locally until sync succeeds or local cache is cleared.
  • Server-side account and log data in Supabase is retained until deletion is requested or otherwise removed by account workflows.

6. Your Rights and Choices

  • You can edit profile fields and meal logs in-app, including updating or deleting diary items.
  • You can sign out at any time, which clears local authenticated session state.
  • You can request account/data deletion or data-access support by contacting us at the email below.
  • You can uninstall the app to remove app data stored locally on your device.

7. Security

Calwise uses authenticated sessions and HTTPS network requests for backend/API communication. Supabase row-level security policies are configured in the project migrations to scope access to user-owned records.

8. Children's Privacy

Calwise is not intended for children under 13. The current backend schema includes age validation constraints aligned with this minimum age.

9. Changes To This Policy

We may update this Privacy Policy from time to time to reflect product or legal changes. The latest version will always be posted on this page.

10. Contact

For privacy questions or concerns, contact us at calwiseapp@gmail.com.
